## Overview

All three roles can add media to Biosafety Land. Media includes images, videos, and documents. This guide shows you how to add media step by step.

### What You Need

- A user account (one of three roles)
- Access to the site at: https://e2e.bsl.staging.cbd.int/en
- Your login email and password
- Media files ready to upload (images, videos, or documents)

### Three User Roles

1. **Contributor** - Can add basic media
2. **Content Manager** - Can add and manage media
3. **Site Manager** - Can add media and manage the whole site

### Supported File Types

**Images:**
- JPG, JPEG (photos)
- PNG (graphics with transparency)
- GIF (animated images)
- SVG (scalable vector graphics)

**Videos:**
- MP4 (most common)
- WebM (web-optimized)
- OGV (open format)

**Documents:**
- PDF (portable documents)
- DOC, DOCX (Word documents)
- XLS, XLSX (Excel spreadsheets)
- TXT (text files)

## Add Media - Contributor Role

Contributors can add new media to the site. Here are the steps.

### Step 1: Log In

1. Go to the website: https://e2e.bsl.staging.cbd.int/en
2. Click the **Log In** button
3. Enter your email: `e2e-contributor@chm-cbd.net`
4. Enter your password
5. Click **Log In**

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-01-login.png -->

### Step 2: Open the Admin Menu

1. Look at the top of the page
2. Find the black toolbar
3. Click on **Tools** in the toolbar

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-02-admin-menu.png -->

### Step 3: Go to Media Section

1. In the toolbar, click **Publishing**
2. A menu will drop down
3. Click on **Media**

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-03-navigate-media.png -->

### Step 4: Click Add Media Button

1. You will see a list of media items
2. Look for the **Add Media** button at the top
3. Click **Add Media**

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-04-add-button.png -->

### Step 5: Choose Media Type

1. A new page will load
2. You will see different media types
3. Choose the type you want to add
4. Click on that media type

Common media types:
- **Image** - For photos and pictures
- **Video** - For video content
- **Document** - For PDF and office files
- **Audio** - For sound files

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-05-choose-type.png -->

### Step 6: Upload Your File

1. Click the **Choose File** button
2. A file browser will open
3. Find your file on your computer
4. Click **Open** to select it
5. The file will start to upload

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-06-upload.png -->

### Step 7: Fill In Media Details

1. Enter a name for your media
2. Add a description (optional)
3. Add alt text if it's an image
4. Fill in any other required fields

**Note:** Alt text helps people who can't see images. It also helps search engines.

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-07-details.png -->

### Step 8: Save the Media

1. Review all the information
2. Click the **Save** button
3. Wait for the save to complete

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-08-save.png -->

### Step 9: Confirm Success

1. A green message will appear
2. It says "Media saved successfully"
3. Your media is now in the system

<!-- SCREENSHOT NEEDED: /images/biosafety-land/media/media-add-contributor-09-success.png -->

**That's it!** You added media as a Contributor.

## Tips and Best Practices

### For All Roles

**File Size Tips:**
- Keep images under 2MB when possible
- Compress large files before upload
- Use web-optimized formats
- Videos should be under 50MB

**Naming Tips:**
- Use clear, descriptive names
- Include date in name: "report-2025-12.pdf"
- Avoid special characters
- Use hyphens not spaces

**Alt Text Tips:**
- Describe what the image shows
- Keep it short (under 125 characters)
- Don't say "image of" or "picture of"
- Be specific and helpful

**Organization Tips:**
- Use consistent naming
- Group related media together
- Add descriptive tags
- Keep media list tidy

### For Contributors

**Before You Upload:**
- Check file type is supported
- Make sure file isn't too large
- Have name and description ready
- Verify file opens correctly

**After You Upload:**
- Double-check the name
- Verify file uploaded fully
- Test the media works
- Note where you saved it

**When Editing:**
- Only edit media you created
- Update description if content changes
- Keep alt text accurate
- Save changes before leaving page

**When Deleting:**
- Only delete media you created
- Make sure media is not in use
- Cannot be undone - be careful
- Double-check before confirming

**When Cloning:**
- Always update the name
- Make clones unique
- Note it's a copy
- Test the clone works

**When Translating:**
- Translate all fields accurately
- Keep same meaning
- Use correct spelling
- Test in target language

### For Content Managers

**Collection Strategy:**
- Create collections by topic
- Use collections for events
- Group by date or season
- Keep collections updated

**Publishing Strategy:**
- Save as unpublished first
- Review before publishing
- Test media displays correctly
- Publish when ready

**Tag Best Practices:**
- Use standard tags across team
- Create tag list for reference
- Don't over-tag items
- Review tags regularly

**When Editing:**
- Check who created the media
- Update tags as needed
- Replace files carefully
- Test after making changes
- Document major updates

**File Replacement:**
- Back up old file first
- Check file name matches
- Verify file type is same
- Test new file works
- Update description if changed

**When Deleting:**
- Check usage first - where is it used?
- Notify team if media is shared
- Update content using this media
- Test affected pages after deletion
- Document what was deleted

**When Cloning:**
- Use clear naming for clones
- Update tags to reflect purpose
- Set correct publish status
- Document why media was cloned
- Test clone before using it

**When Translating:**
- Translate all text fields
- Keep consistent terminology
- Review with native speakers
- Test language-specific files
- Check translations display correctly

### For Site Managers

**Advanced Settings:**
- Set URL aliases for important media
- Use revision log for tracking
- Configure workflows if needed
- Review permissions carefully

**Quality Control:**
- Review media before approval
- Check metadata is complete
- Verify accessibility
- Test across devices

**Media Library Management:**
- Regular cleanup of unused media
- Archive old media files
- Monitor storage usage
- Maintain organization system

**When Editing:**

- Test across all languages

**Revision Management:**

**For All Roles:**

- Know the target language
- Have content ready
- Check required fields
- Understand audience

**Step 2: Translate**
- Use accurate terms
- Keep same meaning
- Follow style guide
- Be culturally aware

**Step 3: Review**
- Check all fields
- Verify spelling
- Test links work
- Ensure quality

**Step 4: Publish**
- Set correct status
- Choose right timing
- Notify team
- Monitor results

### File Type Guidelines

**Use JPG when:**
- You have photos
- File size matters
- You don't need transparency

**Use PNG when:**
- You need transparency
- You have graphics or logos
- Quality is more important than size

**Use SVG when:**
- You have vector graphics
- You need scalability
- You have icons or logos

**Use PDF when:**
- You have documents to share
- You want fixed formatting
- You need printing support

### Common Mistakes to Avoid

**When Adding:**
- Don't upload without checking file type
- Don't use unclear file names
- Don't skip alt text for images
- Don't upload oversized files
- Don't forget to save your work

**When Editing:**
- Don't edit without a backup
- Don't replace files without checking
- Don't skip the revision log
- Don't forget to test changes
- Don't edit media in use without notice

**When Deleting:**
- Don't delete without checking usage
- Don't skip the warning messages
- Don't delete shared media without notice
- Don't forget it's permanent
- Don't delete without a backup plan

**When Cloning:**
- Don't keep same name as original
- Don't clone without purpose
- Don't forget to update fields
- Don't create excessive clones
- Don't skip testing cloned media

**When Translating:**
- Don't use machine translation only
- Don't skip fields
- Don't forget to test
- Don't ignore cultural context
- Don't translate file names literally

**Do:**
- Check file before upload or replacement
- Use descriptive names
- Always add alt text
- Optimize file sizes
- Save and verify success
- Document your changes
- Test before and after edits
- Review usage before deleting
- Back up important media
- Communicate with your team
- Update clones appropriately
- Keep library organized
- Translate accurately
- Review translations with experts

### When to Use Each Media Type

**Images:**
- Photos of events
- Diagrams and charts
- Screenshots and tutorials
- Logos and branding

**Videos:**
- Tutorials and demos
- Event recordings
- Announcements
- Training materials

**Documents:**
- Reports and forms
- Policy documents
- Meeting notes
- Reference materials

**Audio:**
- Podcasts
- Interviews
- Announcements
- Music files

### When to Delete vs. Unpublish

**Unpublish when:**
- Media might be needed later
- It's temporarily unused
- It's seasonal content
- You want to keep the history
- Other content might reference it
- It's part of an archive

**Delete when:**
- Media is duplicate
- File is corrupt or wrong
- It's definitely not needed
- Storage space is limited
- It's a test file
- Privacy requires removal

**Best Practice:** Unpublish first. Wait. Then delete if still unneeded.

### Safe Deletion Checklist

Before deleting media:
- [ ] Check where media is used
- [ ] Review all content references
- [ ] Notify team if shared
- [ ] Back up if important
- [ ] Update content using it
- [ ] Test affected pages
- [ ] Document the deletion
- [ ] Monitor for broken links

**Remember:** Deletion is permanent. There is no undo.

### When to Clone vs. Create New

**Clone when:**
- You need a similar media item
- You want to keep same settings
- Creating variations (sizes, crops)
- Making translated versions
- Testing changes safely
- Building on existing work

**Create New when:**
- Media is completely different
- You want fresh metadata
- Starting from scratch
- Different file type needed
- No relationship to existing
- Cleaner organization desired

**Best Practice:** Clone saves time. But don't clone everything. Keep library organized.

### Common Cloning Mistakes to Avoid

**Don't:**
- Keep original name on clone
- Clone without updating fields
- Create too many clones
- Forget to update description
- Skip testing the clone
- Clone everything unnecessarily

**Do:**
- Use clear naming for clones
- Update all relevant fields
- Clean up unused clones
- Document clone purpose
- Test before using
- Only clone when beneficial
