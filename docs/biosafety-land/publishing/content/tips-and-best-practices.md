## Tips and Best Practices

### Before You Add Content

1. **Plan your content**
   - Know what you want to say
   - Gather images and files first
   - Check if similar content exists

2. **Choose the right type**
   - Use Article for news
   - Use Page for general info
   - Use Document for downloads
   - Use Event for meetings

3. **Prepare your text**
   - Write clear titles
   - Use short sentences
   - Break text into paragraphs
   - Check spelling

### While Adding Content

1. **Fill required fields first**
   - Look for fields marked with *
   - These must be filled in
   - The form won't save without them

2. **Add good descriptions**
   - Describe your images
   - This helps people who use screen readers
   - It also helps with search

3. **Use categories and tags**
   - Add relevant tags
   - Choose the right category
   - This helps people find your content

4. **Preview before saving**
   - Look at how it will appear
   - Check for mistakes
   - Make sure images show correctly

### After Adding Content

1. **Review your work**
   - Read through the content
   - Check all links work
   - Make sure images load

2. **Test different devices**
   - Check on mobile phones
   - Check on tablets
   - Check on computers

3. **Share with others**
   - Tell your team about new content
   - Link to it from other pages
   - Promote it if needed

### Common Mistakes to Avoid

❌ **Don't:**

- Leave required fields empty
- Use very long titles
- Forget to add images descriptions
- Publish without reviewing
- Use all capital letters
- Put too much text in one paragraph

✅ **Do:**

- Fill all required fields
- Keep titles short and clear
- Add image descriptions
- Review before publishing
- Use normal capitalization
- Break text into short paragraphs

### Writing Good Content

1. **Keep it simple**
   - Use short words
   - Use short sentences
   - One idea per sentence

2. **Make it clear**
   - Say what you mean
   - Don't use jargon
   - Explain technical terms

3. **Make it easy to read**
   - Use headings
   - Use bullet points
   - Add white space
   - Keep paragraphs short

4. **Make it accessible**
   - Add image descriptions
   - Use clear link text
   - Don't rely only on color
   - Use good contrast

### Best Practices for Editing Content

1. **Before you edit**
   - Read the current content first
   - Know what needs to change
   - Check if you have permission to edit
   - Make a note of major changes

2. **While editing**
   - Change only what needs updating
   - Don't remove important information
   - Keep the writing style consistent
   - Check that links still work
   - Test images and media

3. **Always add revision notes**
   - Write clear notes about changes
   - Say what you changed and why
   - This helps your team understand updates
   - Makes it easy to track history

Good revision note examples:

- "Updated event date from June to July"
- "Added three new images of the facility"
- "Fixed broken links in the footer"
- "Corrected spelling errors in paragraphs 2 and 3"

Bad revision note examples:

- "Updates" (too vague)
- "Changes" (doesn't say what changed)
- "" (empty - always write something)

4. **Review before saving**
   - Read through all your changes
   - Preview how it will look
   - Check for typos
   - Make sure formatting is correct
   - Verify images display properly

5. **After editing**
   - View the live content
   - Test all links
   - Check on different devices
   - Tell relevant team members
   - Monitor for any issues

### Common Editing Mistakes to Avoid

❌ **Don't:**

- Edit without reading the current content
- Make changes without adding revision notes
- Delete content without checking with others
- Change URLs without creating redirects
- Remove images that are used elsewhere
- Edit during high traffic times (if possible)
- Make too many changes at once

✅ **Do:**

- Read the content before editing
- Always add clear revision notes
- Check with team before major deletions
- Keep old URLs or set up redirects
- Check image usage before removing
- Edit during quiet times when possible
- Make changes in logical groups

### When to Edit vs. Create New

**Edit existing content when:**

- Information needs updating
- Fixing errors or typos
- Adding small amounts of new info
- Updating dates or times
- Replacing old images
- Fixing broken links

**Create new content when:**

- Covering a completely new topic
- Major restructuring needed
- Different target audience
- Different purpose or goal
- Old content should stay archived

### Best Practices for Deleting Content

1. **Before you delete**
   - Check where the content is used
   - Save a backup if it's important
   - Tell your team about the deletion
   - Consider unpublishing instead
   - Plan redirects for the URL

2. **Things to check**
   - Is content linked from other pages?
   - Is content in any menus?
   - Do other sites link to this page?
   - Is this content part of a series?
   - Are there translations?

3. **When to delete vs. unpublish**

**Delete when:**

- Content is completely wrong
- Content violates policies
- Content is duplicate
- Content is test data
- You're sure you won't need it

**Unpublish when:**

- Content might be needed later
- Content is seasonal
- Content needs major updates
- You're not sure about deleting
- Content has historical value

4. **After deleting**
   - Fix broken links
   - Update menus
   - Set up redirects
   - Tell your team
   - Monitor for problems

5. **Deletion warnings**

⚠️ **Remember:**

- Deletion cannot be undone
- Links to deleted content will break
- Search engines may show 404 errors
- Users may have bookmarks that break
- Always check before deleting

✅ **Safe deletion checklist:**

- [ ] Checked where content is used
- [ ] Saved backup if needed
- [ ] Notified team members
- [ ] Planned URL redirects
- [ ] Updated related content
- [ ] Removed from menus
- [ ] Added deletion note

### Best Practices for Cloning Content

1. **When to clone content**
   - Creating similar content saves time
   - Making templates for regular posts
   - Creating seasonal variations
   - Testing changes before editing original
   - Making content for different audiences

2. **Before you clone**
   - Check the original content is correct
   - Remove outdated information first
   - Update images if needed
   - Check for broken links
   - Verify author information

3. **What to change after cloning**

**Always change:**

- Title - Remove "(Clone)" text
- URL - Make it unique
- Publish date - Set to current date
- Meta description - Make it unique
- Any specific names or dates

**Review and update:**

- Body text - Check for relevance
- Images - Update if needed
- Links - Verify they work
- Categories - Are they still right?
- Tags - Update for new content

4. **Clone vs. Create New**

**Clone when:**

- Content structure is similar
- You need the same format
- It saves significant time
- Creating a template
- Making slight variations

**Create new when:**

- Content is completely different
- You need a different structure
- Original has many errors
- You want a fresh start
- Different content type needed

5. **Common clone mistakes**

❌ **Don't:**

- Forget to change the title
- Leave "(Clone)" in the title
- Keep the same URL
- Clone content with errors
- Clone outdated information
- Forget to update dates
- Leave old author information

✅ **Do:**

- Give it a unique title
- Set a new URL
- Update all dates
- Review all content
- Update author if needed
- Check links still work
- Test before publishing

6. **Tips for different roles**

**Contributors:**

- Can clone any published content (own or others')
- Can also clone own drafts
- Always change title and URL
- Review all cloned content
- Save as draft first
- Ask manager if unsure

**Content Managers:**

- Clone any content carefully
- Update author if cloning others' work
- Set proper workflow status
- Configure language settings
- Test in preview mode

**Site Managers:**

- Use advanced clone options wisely
- Set permissions correctly
- Configure URL redirects if needed
- Document why you cloned
- Test all functionality

7. **After cloning**
   - Review all fields carefully
   - Test all links
   - Preview before publishing
   - Update any timestamps
   - Check mobile view
   - Verify it works correctly

✅ **Safe cloning checklist:**

- [ ] Original content is correct
- [ ] New title is unique
- [ ] New URL is set
- [ ] Dates are updated
- [ ] Author is correct
- [ ] Images are appropriate
- [ ] Links are verified
- [ ] Categories and tags reviewed
- [ ] Previewed before publishing
- [ ] Tested on mobile

### Quick Reference

**To add content:**

1. Log in
2. Click Publishing > Content
3. Click Add Content
4. Choose content type
5. Fill in the form
6. Save or publish

**To edit content:**

1. Log in
2. Click Publishing > Content
3. Find the content in the list
4. Click Edit button
5. Make your changes
6. Add revision note
7. Click Save

**To delete content:**

1. Log in
2. Click Publishing > Content
3. Find the content in the list
4. Click Delete button
5. Read the warning
6. Add deletion note (if required)
7. Confirm deletion
8. Clean up broken links (if needed)

**To clone content:**

1. Log in
2. Click Publishing > Content
3. Find the content to clone
4. Click Operations > Clone
5. Review cloned fields
6. Change title (remove "(Clone)")
7. Update URL and dates
8. Make necessary changes
9. Save or publish
