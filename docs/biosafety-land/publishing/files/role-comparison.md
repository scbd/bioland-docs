## Role Comparison

This table shows what each role can do with files.

| Action | Contributor | Content Manager | Site Manager |
|--------|:-----------:|:---------------:|:------------:|
| **Access Files tab** | ❌ No | ❌ No | ✅ Yes |
| **View files** | ❌ No | ❌ No | ✅ Yes |
| **Add files** | ❌ No | ❌ No | ✅ Yes |
| **Edit files** | ❌ No | ❌ No | ✅ Yes |
| **Replace file** | ❌ No | ❌ No | ✅ Yes |
| **Delete files** | ❌ No | ❌ No | ✅ Yes |
| **Configure advanced settings** | ❌ No | ❌ No | ✅ Yes |
| **View file usage** | ❌ No | ❌ No | ✅ Yes |
| **Manage all files** | ❌ No | ❌ No | ✅ Yes |

### Key Differences

**Contributor:**
- Cannot access file management at all
- Navigating to Publishing → Files shows "Access Denied"

**Content Manager:**
- Cannot access file management
- Navigating to Publishing → Files shows "Access Denied"
- Despite having broader content permissions, file management is restricted to Site Managers

**Site Manager:**
- Only role with access to the Files tab
- Can add, edit, replace, and delete files
- Can configure advanced settings (authoring info, URL aliases, revisions)
- Can see where files are used across the site
