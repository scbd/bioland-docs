# Media Management - Tips and Best Practices

This page provides helpful tips for working with media in Biosafety Land.

[[toc]]

---

## For All Roles

### File Size Tips
- Keep images under 2MB when possible
- Compress large files before upload
- Use web-optimized formats
- Videos should be under 50MB

### Naming Tips
- Use clear, descriptive names
- Include date in name: "report-2025-12.pdf"
- Avoid special characters
- Use hyphens not spaces

### Alt Text Tips
- Describe what the image shows
- Keep it short (under 125 characters)
- Don't say "image of" or "picture of"
- Be specific and helpful

### Organization Tips
- Use consistent naming
- Group related media together
- Add descriptive tags
- Keep media list tidy

## For Contributors

### Before You Upload
- Check file type is supported
- Make sure file isn't too large
- Have name and description ready
- Verify file opens correctly

### After You Upload
- Double-check the name
- Verify file uploaded fully
- Test the media works
- Note where you saved it

### When Editing
- Only edit media you created
- Update description if content changes
- Keep alt text accurate
- Save changes before leaving page

### When Deleting
- Only delete media you created
- Make sure media is not in use
- Cannot be undone - be careful
- Double-check before confirming

### When Cloning
- Always update the name
- Make clones unique
- Note it's a copy
- Test the clone works

### When Translating
- Translate all fields accurately
- Keep same meaning
- Use correct spelling
- Test in target language

## For Content Managers

- Create collections by topic
- Use collections for events
- Group by date or season
- Keep collections updated

### Publishing Strategy
- Save as unpublished first
- Review before publishing
- Test media displays correctly
- Publish when ready

### Tag Best Practices
- Use standard tags across team
- Create tag list for reference
- Don't over-tag items
- Review tags regularly

### When Editing
- Check who created the media
- Update tags as needed
- Replace files carefully

### When Cloning

### When Translating

## Translation Workflow Guidance

### For All Roles

**Step 1: Prepare**
- Know the target language
- Have content ready
- Check required fields
- Understand audience

**Step 2: Translate**
- Use accurate terms
- Keep same meaning
- Follow style guide
- Be culturally aware

**Step 3: Review**
- Check all fields
- Verify spelling
- Test links work
- Ensure quality

**Step 4: Publish**
- Set correct status
- Choose right timing
- Notify team
- Monitor results

### Language-Specific Files
- **When to use:** Document contains language-specific text
- **Examples:** PDFs with text, videos with subtitles
- **Best practice:** Upload for each language
- **Alternative:** Use same file if language-neutral (like music or photos)

## File Type Guidelines

### Use JPG when:
- You have photos
- File size matters
- You don't need transparency

### Use PNG when:
- You need transparency
- You have graphics or logos
- Quality is more important than size

### Use SVG when:
- You have vector graphics
- You need scalability
- You have icons or logos

### Use PDF when:
- You have documents to share
- You want fixed formatting
- You need printing support

## Common Mistakes to Avoid

### When Adding
- Don't upload without checking file type
- Don't use unclear file names
- Don't skip alt text for images
- Don't upload oversized files
- Don't forget to save your work

### When Editing
- Don't edit without a backup
- Don't replace files without checking
- Don't skip the revision log
- Don't forget to test changes
- Don't edit media in use without notice

### When Deleting
- Don't delete without checking usage
- Don't skip the warning messages
- Don't delete shared media without notice
- Don't forget it's permanent
- Don't delete without a backup plan

### When Cloning
- Don't keep same name as original
- Don't clone without purpose
- Don't forget to update fields
- Don't create excessive clones
- Don't skip testing cloned media

### When Translating
- Don't use machine translation only
- Don't skip fields
- Don't forget to test
- Don't ignore cultural context
- Don't translate file names literally

### Do's
- Check file before upload or replacement
- Use descriptive names
- Always add alt text
- Optimize file sizes
- Save and verify success
- Document your changes
- Test before and after edits
- Review usage before deleting
- Back up important media
- Communicate with your team
- Update clones appropriately
- Keep library organized
- Translate accurately
- Review translations with experts

## When to Delete vs. Unpublish

### Unpublish when:
- Media might be needed later
- It's temporarily unused
- It's seasonal content
- You want to keep the history
- Other content might reference it
- It's part of an archive

### Delete when:
- Media is duplicate
- File is corrupt or wrong
- It's definitely not needed
- Storage space is limited
- It's a test file
- Privacy requires removal

**Best Practice:** Unpublish first. Wait. Then delete if still unneeded.

---

## Safe Deletion Checklist

Before deleting media:
- [ ] Check where media is used
- [ ] Review all content references
- [ ] Notify team if shared
- [ ] Back up if important
- [ ] Update content using it
- [ ] Test affected pages
- [ ] Document the deletion
- [ ] Monitor for broken links

**Remember:** Deletion is permanent. There is no undo.

---

## When to Clone vs. Create New

### Clone when:
- You need a similar media item
- You want to keep same settings
- Creating variations (sizes, crops)
- Making translated versions
- Testing changes safely
- Building on existing work

### Create New when:
- Media is completely different
- You want fresh metadata
- Starting from scratch
- Different file type needed
- No relationship to existing
- Cleaner organization desired

**Best Practice:** Clone saves time. But don't clone everything. Keep library organized.

---

## Common Cloning Mistakes to Avoid

### Don't:
- Keep original name on clone
- Clone without updating fields
- Create too many clones
- Forget to update description
- Skip testing the clone
- Clone everything unnecessarily

### Do:
- Use clear naming for clones
- Update all relevant fields
- Clean up unused clones
- Document clone purpose
- Test before using
- Only clone when beneficial
